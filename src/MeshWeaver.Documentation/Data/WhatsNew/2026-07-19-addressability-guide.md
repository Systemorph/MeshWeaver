# New guide — Addressability of Objects

A new documentation page, **Addressability of Objects**, walks through the idea at the heart
of the platform: everything — a record, its schema, the data model behind it, its views, its
files, its access, even the agent that reasons over it — is a node in one addressable mesh, and
each of those facets is reachable by a path.

Following a single request end to end, it shows how each step is the same move — *name the object,
then act on it* — and how that one property is what lets the assistant navigate to where you are,
read the data and the schema that matches the database, resolve dimension values by search, rank
results by meaning, and reach the very files you mention by name. Find it under **Data Mesh →
Addressability of Objects**.
